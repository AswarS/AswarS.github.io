java | 错误：GBK编码GBK的不可映射字符

​ 此错误的产生是因为在编写java时使用了中文，并且源文件的编码格式不是ANSI编码格式。

​ java在编译时需要使用JDK开发包中的javac.exe命令，JDK语言为国际版，默认编码格式为UNICODE。在未指定源文件的编码格式情况下，JDK会优先获取操作系统的file.encoding参数作为源文件的编码格式，然后JDK就把我们的JAVA源程序从file.encoding编码格式转化为JAVA内部默认的UNICODE格式放到内存中。然后，javac把转换后的UNICODE格式的文件进行编译成class类文件，此时.class文件是UNICODE编码的，它暂放在内存中，之后，JDK将此以UNICODE编码编译后的.class文件保存到操作系统中形成我们见到的.class文件。

解决办法：

  1. 编译时使用命令 javac -encoding utf-8(此处为源文件的编码格式) xx.java
  2. 将编辑器的默认编码格式改为ANSI
